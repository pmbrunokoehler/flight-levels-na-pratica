# Template — Checklist de Cadências (Flight Level 2)

> Use este checklist para desenhar cada cadência antes de colocá-la na agenda. Se uma reunião não passa no teste da "pergunta única", ela não deveria existir.

## Teste de validação de uma cadência

Antes de criar qualquer cadência, responda:

- [ ] Qual é a **pergunta única** que só esta cadência responde?
- [ ] Quem é o **público mínimo** necessário? (Cada pessoa a mais precisa de justificativa.)
- [ ] Qual a **duração máxima**? (Curta por padrão; estenda apenas com motivo.)
- [ ] Que **decisão** deve sair daqui? (Se a resposta for "nenhuma, é só informe", reconsidere a reunião.)

| Campo | Definição |
|-------|-----------|
| **Pergunta única** | O que está bloqueando o fluxo entre os times agora? |
| **Frequência** | 📝 [PREENCHER — ex.: semanal] |
| **Duração** | 📝 [PREENCHER — ex.: 30-45 min] |
| **Participantes** | 📝 [PREENCHER — ex.: representante de cada squad + produto + engenharia] |
| **Foco** | Itens que cruzam times; dependências; decisões de desbloqueio |
| **Fora de escopo** | Assuntos internos de um único time |
| **Saída esperada** | Dependências destravadas, prioridades ajustadas |

## Sinais de que uma cadência precisa de ajuste

- A reunião virou rodada de status individual em vez de coordenação.
- Pessoas presentes que não participam de nenhuma decisão.
- Estende-se sempre além do tempo previsto.
- Nenhuma decisão sai dela com regularidade.
- Discute-se assunto interno de um time só.
